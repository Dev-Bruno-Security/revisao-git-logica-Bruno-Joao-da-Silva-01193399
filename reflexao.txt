- A lógica dedutiva no Git aparece de forma clara no dia a dia, partimos de premissas gerais, todo commit tem um hash único, branches são ponteiros móveis, merge combina históricos e chegamos a conclusões específicas sobre o estado do código,
- por exemplo, se sei que o ponteiro head está na branch main e que o último commit é o X, deduzo necessariamente que o conteúdo atual dos arquivos é exatamente o estado salvo no commit X, quando faço git checkout de um commit antigo, 
- deduzo com certeza qual será o conteúdo dos arquivos porque o histórico é imutável.
- A decomposição é um dos pilares do Git e se manifesta especialmente no uso de branches, um projeto grande é decomposto em pequenos pedaços independentes de trabalho features, correções, experimentos, 
-Cada branch representa uma unidade lógica isolada que pode ser desenvolvida, testada e revisada sem afetar o código estável da branch main, Quando criamos uma branch, estamos aplicando o princípio 
“dividir para conquistar”, reduzimos a complexidade do problema total em problemas menores e gerenciáveis, o merge posterior recompõe essas partes de forma controlada, mantendo a coerência do histórico.
